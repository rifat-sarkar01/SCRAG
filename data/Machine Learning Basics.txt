1. INTRODUCTION TO MACHINE LEARNING
================================================================================

Definition:
Machine Learning (ML) is a subset of Artificial Intelligence (AI) that enables
systems to learn and improve from experience (data) without being explicitly
programmed.

The Core Paradigm:
Instead of writing a program with explicit rules (e.g., "if X, then Y"), we
provide a model with data and an algorithm to find the rules itself.

Types of Machine Learning:
-------------------------
1. Supervised Learning: The model learns from labeled data.
   - Goal: Map input (X) to output (Y) based on example pairs.
   - Types:
     a. Classification: Predicting a discrete category (e.g., Spam/Not Spam).
     b. Regression: Predicting a continuous value (e.g., House Price).

2. Unsupervised Learning: The model finds patterns in unlabeled data.
   - Goal: Discover the underlying structure or distribution of the data.
   - Types:
     a. Clustering: Grouping data points (e.g., Customer Segmentation).
     b. Dimensionality Reduction: Reducing the number of features (e.g., PCA).

3. Reinforcement Learning: An agent learns to make decisions by interacting with
   an environment to maximize a reward. (Not covered in detail here).

2. CORE CONCEPTS & TERMINOLOGY
================================================================================

- Features (X): The input variables used for prediction. (e.g., square footage,
  number of bedrooms).
- Target (y): The output variable we are trying to predict. (e.g., House Price).
- Training Set: The subset of data used to train the model.
- Test Set: The subset of data used to evaluate the model's performance on
  unseen data. (Typically 70-80% train, 20-30% test).
- Validation Set: A separate subset used to tune the model's hyperparameters.
- Model: The mathematical representation of the relationship between features
  and the target.
- Hypothesis (h): The specific function that the model learns (e.g., h(x) = mx + b).
- Cost/Loss Function: A function that measures how wrong the model's predictions
  are. The goal is to minimize this.
- Gradient Descent: An optimization algorithm used to minimize the cost function
  by iteratively updating the model's parameters.
- Bias: Error introduced by approximating a real-world problem with a simplified
  model. (High bias = Underfitting).
- Variance: Error introduced by the model's sensitivity to small fluctuations in
  the training data. (High variance = Overfitting).

The Bias-Variance Trade-off:
---------------------------
- Underfitting: Model is too simple (High Bias). Fails to capture the pattern in
  the training data. Poor performance on train and test data.
- Overfitting: Model is too complex (High Variance). Learns the noise in the
  training data. Excellent performance on train, poor on test.
- Goal: Find the "Sweet Spot" in the middle.

3. DATA PREPROCESSING
================================================================================

Real-world data is messy. Preprocessing is crucial for model performance.

Common Steps:
------------
1. Handling Missing Values:
   - Imputation: Fill missing values with mean, median, mode, or a constant.
   - Deletion: Remove rows or columns with missing data (use cautiously).

2. Encoding Categorical Data:
   - Machines work with numbers.
   - Label Encoding: Assign a unique integer to each category (e.g., Red=0,
     Green=1, Blue=2). Suitable for ordinal data.
   - One-Hot Encoding: Create binary (0/1) columns for each category. Suitable
     for nominal data (e.g., Color_Red, Color_Green, Color_Blue).

3. Feature Scaling:
   - Normalization (Min-Max Scaling): Scales values to a range of [0, 1].
     Formula: (x - min) / (max - min).
   - Standardization (Z-score Scaling): Scales values to have a mean of 0 and
     standard deviation of 1. Formula: (x - mean) / std.
   - Why? Algorithms like SVM, KNN, and Linear Regression are sensitive to the
     magnitude of features.

4. Splitting the Dataset:
   - `X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)`

5. Handling Imbalanced Data (For Classification):
   - If one class has significantly more samples than another, the model will be
     biased.
   - Techniques: Oversampling (e.g., SMOTE), Undersampling, or using class weights.

4. MODEL EVALUATION & VALIDATION
================================================================================

How do we know if a model is good?

Common Metrics:
--------------
1. Classification Metrics:
   - Confusion Matrix: A table showing True Positives, True Negatives, False
     Positives, False Negatives.
   - Accuracy: (TP + TN) / (TP + TN + FP + FN). Good for balanced datasets.
   - Precision: TP / (TP + FP). Out of all positive predictions, how many were
     correct? (Relevant to minimizing False Positives).
   - Recall (Sensitivity): TP / (TP + FN). Out of all actual positives, how many
     did we catch? (Relevant to minimizing False Negatives).
   - F1-Score: The harmonic mean of Precision and Recall.
   - ROC-AUC: Measures the model's ability to distinguish between classes across
     all thresholds.

2. Regression Metrics:
   - Mean Absolute Error (MAE): Average of the absolute differences.
   - Mean Squared Error (MSE): Average of the squared differences. Penalizes
     large errors more.
   - Root Mean Squared Error (RMSE): The square root of MSE. Interpretable in
     the same units as the target.
   - R-squared (R²): Coefficient of determination. Represents the proportion of
     variance in the target that can be explained by the features. (0 to 1).

Cross-Validation:
----------------
- A robust technique to evaluate a model by splitting the data into 'k' folds.
- The model is trained on `k-1` folds and tested on the remaining fold.
- Process is repeated `k` times, and the average performance is reported.
- K-Fold CV is standard. Stratified K-Fold ensures each fold has the same
  proportion of class labels for classification.

5. SUPERVISED LEARNING - REGRESSION
================================================================================

Goal: Predict a continuous target variable.

1. Linear Regression:
   - Assumes a linear relationship between features and target.
   - Equation: y = b0 + b1*x1 + b2*x2 + ... + bn*xn.
   - Cost Function: Mean Squared Error (MSE).
   - Optimization: Gradient Descent or Ordinary Least Squares.
   - Pros: Simple, fast, interpretable.
   - Cons: Assumes linearity, sensitive to outliers.

2. Polynomial Regression:
   - Extends Linear Regression by adding polynomial terms (e.g., x^2, x^3).
   - Allows the model to fit non-linear data.

3. Regularization (For Linear Models):
   - Techniques to prevent overfitting by adding a penalty term to the cost
     function.
   - Ridge Regression (L2 Regularization): Penalty = sum of squared coefficients.
     Helps shrink coefficients but doesn't eliminate them.
   - Lasso Regression (L1 Regularization): Penalty = sum of absolute values of
     coefficients. Can shrink some coefficients to zero (feature selection).

================================================================================
6. SUPERVISED LEARNING - CLASSIFICATION
================================================================================

Goal: Predict a discrete class label.

1. Logistic Regression:
   - Despite the name, it's a classification algorithm.
   - Uses the Sigmoid function to output a probability between 0 and 1.
   - Decision boundary is linear.

2. K-Nearest Neighbors (KNN):
   - Lazy learner: Stores all training data. Predicts by finding the 'K' nearest
     neighbors to the new data point and taking a majority vote.
   - Requires feature scaling.
   - Pros: Simple, no training phase.
   - Cons: Slow for large datasets, sensitive to the value of 'K'.

3. Naive Bayes:
   - Based on Bayes' Theorem with the "naive" assumption of feature independence.
   - Fast and works well with high-dimensional data.
   - Types: GaussianNB (continuous data), MultinomialNB (discrete data).

4. Support Vector Machines (SVM):
   - Finds the optimal hyperplane that best separates the classes by maximizing
     the margin.
   - Can handle non-linear data using the "Kernel Trick" (e.g., RBF, Polynomial).
   - Very powerful, works well on high-dimensional data.
   - Cons: Not interpretable, memory-intensive for large datasets.

5. Decision Trees:
   - Uses a tree-like model of decisions. Splits data based on feature values.
   - Splitting criteria: Gini Impurity or Entropy (Information Gain).
   - Pros: Highly interpretable, no need for scaling.
   - Cons: Prone to overfitting.

7. UNSUPERVISED LEARNING - CLUSTERING
================================================================================

Goal: Discover inherent groupings in the data.

1. K-Means Clustering:
   - Partitions data into 'K' clusters.
   - Algorithm:
     1. Initialize 'K' centroids randomly.
     2. Assign each data point to the nearest centroid.
     3. Recalculate centroids as the mean of all points in the cluster.
     4. Repeat steps 2 and 3 until convergence.
   - Choosing 'K': Elbow Method (plotting inertia against 'K').
   - Requires feature scaling.

2. Hierarchical Clustering:
   - Builds a hierarchy of clusters.
   - Agglomerative (Bottom-up): Each point starts as its own cluster, merges the
     closest clusters iteratively.
   - Divisive (Top-down): Starts with one cluster and splits it.
   - Output is a Dendrogram.

3. DBSCAN (Density-Based Spatial Clustering of Applications with Noise):
   - Clusters based on density. Finds clusters of arbitrary shape.
   - Can identify outliers (noise points).
   - Does not require specifying the number of clusters.

================================================================================
8. UNSUPERVISED LEARNING - DIMENSIONALITY REDUCTION
================================================================================

Goal: Reduce the number of features while preserving as much information as possible.

1. Principal Component Analysis (PCA):
   - Transforms the data into a new coordinate system where the greatest
     variance lies on the first axis (Principal Component), second greatest on
     the second, and so on.
   - Used for visualization, noise reduction, and speeding up other algorithms.

2. t-SNE (t-Distributed Stochastic Neighbor Embedding):
   - Non-linear technique primarily used for visualization in 2D or 3D.
   - Excellent at preserving local structure.

9. INTERMEDIATE TOPICS
================================================================================

Ensemble Methods: Combining multiple models to get a better result.

1. Bagging (Bootstrap Aggregating):
   - Trains multiple models in parallel on different subsets of the data
     (bootstrapped samples).
   - Reduces variance and helps prevent overfitting.
   - Example: Random Forest.

2. Random Forest:
   - An ensemble of Decision Trees.
   - Each tree is trained on a bootstrapped sample of the data and a random
     subset of features.
   - Prediction is the majority vote (classification) or average (regression).

3. Boosting:
   - Trains models sequentially. Each new model attempts to correct the errors
     of the previous one.
   - Reduces bias and variance.
   - Example: AdaBoost, Gradient Boosting (XGBoost, LightGBM, CatBoost).

4. XGBoost (Extreme Gradient Boosting):
   - An optimized distributed gradient boosting library.
   - Very popular in competitions for its performance and speed.
   - Handles missing values well.

10. THE MACHINE LEARNING WORKFLOW
================================================================================

A typical end-to-end ML project looks like this:

1. Define the Problem: What business problem are you solving? Is it
   classification or regression?

2. Gather Data: Collect the data from databases, APIs, or files.

3. Data Cleaning & Preprocessing:
   - Handle missing values, outliers, and errors.
   - Encode categorical variables.

4. Exploratory Data Analysis (EDA):
   - Visualize the data to understand relationships and distributions.
   - Create new features (Feature Engineering).

5. Feature Engineering & Selection:
   - Create new features from existing ones.
   - Select the most relevant features to reduce noise.

6. Train/Test Split: Divide the data into training, validation, and test sets.

7. Model Selection & Training:
   - Choose a base model (e.g., Linear Regression, SVM).
   - Train it on the training set.

8. Hyperparameter Tuning:
   - Use Grid Search or Random Search on the validation set to find the best
     hyperparameters.

9. Model Evaluation:
   - Evaluate the final model on the *test set* (unseen data) using appropriate
     metrics.

10. Deployment & Monitoring:
    - Deploy the model to a production environment.
    - Monitor its performance and retrain periodically.

11. GLOSSARY OF IMPORTANT TERMS
================================================================================

- Feature: An individual measurable property of a phenomenon being observed.
- Target: The variable we are trying to predict.
- Epoch: One complete pass through the entire training dataset.
- Batch: A subset of the training dataset used in one iteration of gradient descent.
- Learning Rate: A hyperparameter that controls how much to change the model in
  response to the estimated error each time the model weights are updated.
- Hyperparameter: A parameter whose value is set before the learning process
  begins (e.g., 'K' in KNN, learning rate).
- Parameter: A variable whose value is learned from the data (e.g., weights in
  Linear Regression).
- Kernel Trick: A method used in SVM to implicitly transform data into a
  higher-dimensional space without actually computing the transformation.
